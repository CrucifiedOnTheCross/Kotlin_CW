# Тестовые сценарии для приложения "Система учета клиентов автомойки"

| № | Действие | Ожидаемый результат |
|---|---|---|
| 1 | Запустить приложение при наличии корректного `car_wash_clients.json` | Данные загружаются, выводится количество записей |
| 2 | Запустить приложение без файла данных | Приложение сообщает об отсутствии файла и открывает пустой список |
| 3 | Выбрать пункт "Показать всех клиентов" при пустой коллекции | Выводится сообщение "Список клиентов пуст" |
| 4 | Добавить клиента с корректными данными | Запись добавляется, ей назначается ID |
| 5 | Ввести текст вместо числа в полях посещений, суммы или скидки | Приложение сообщает об ошибке и просит повторить ввод |
| 6 | Редактировать клиента по существующему ID | Данные клиента обновляются |
| 7 | Редактировать клиента по несуществующему ID | Выводится сообщение, что клиент не найден |
| 8 | Удалить клиента по существующему ID | Запись удаляется из коллекции |
| 9 | Удалить клиента по несуществующему ID | Выводится сообщение, что клиент не найден |
| 10 | Найти клиента по части ФИО, телефона, госномера или модели | Выводятся подходящие записи |
| 11 | Выполнить поиск по отсутствующему значению | Выводится сообщение "Совпадения не найдены" |
| 12 | Отсортировать клиентов по общей сумме оплат | Записи выводятся в выбранном порядке |
| 13 | Отсортировать клиентов по дате последнего визита | Записи выводятся в порядке дат |
| 14 | Показать агрегированные показатели | Выводятся количество клиентов, общая выручка, средняя сумма оплат и клиент с максимумом оплат |
| 15 | Сохранить данные | Файл `car_wash_clients.json` создается или обновляется корректным JSON |
